Voici les directives techniques strictes à appliquer sur vos infrastructures pour combler les failles découvertes :

    {% if detect.ffuf %}
  • [URGENCE] Compromission d'identifiants / Fichiers sensibles exposés :
    Des informations critiques (ex: identifiants de base de données) sont accessibles publiquement.
    • Action immédiate : Procédez instantanément à la rotation des mots de passe exposés.
    • Correction : Déplacez les fichiers d'environnement (`.env`, `config.bak`) en dehors du répertoire racine du serveur Web (`/var/www/html/`). Restreignez leurs permissions avec `chmod 600`.
    • {% endif %} {% if detect.nmap %}
  • [RÉSEAU] Exposition de services d'administration (Ports ouverts) :
    Des ports sensibles (ex: SSH, BDD) ont été détectés comme ouverts sur l'Internet public.
    • Correction : Configurez votre pare-feu (ex: `ufw` ou `iptables`) pour bloquer l'accès à ces ports par défaut.
    • N'autorisez les connexions SSH (Port 22) que via une liste blanche d'IP statiques (Whitelisting) ou un VPN d'entreprise. Désactivez l'authentification par mot de passe au profit de clés cryptographiques (`PasswordAuthentication no` dans `sshd_config`).
    • {% endif %} {% if detect.config or detect.outdated %}
  • [SERVEUR] Défauts de configuration et Obsolescence :
    Le serveur divulgue sa version technique ou utilise des composants obsolètes.
    • Correction : Appliquez les derniers patchs de sécurité sur votre serveur Web et vos moteurs (PHP, Apache, Nginx).
    • Masquez la signature du serveur en modifiant `ServerTokens Prod` et `ServerSignature Off` dans la configuration Apache, ou `server_tokens off;` sous Nginx.
    • Supprimez ou protégez par mot de passe les répertoires listés dans le fichier `robots.txt`.
    • {% endif %} {% if detect.ssl %}
  • [CRYPTOGRAPHIE] Certificats SSL/TLS Invalides ou Faibles :
    La chaîne de confiance cryptographique est brisée (certificat auto-signé, expiré ou Suites cryptographiques faibles).
    • Correction : Remplacez le certificat actuel par un certificat valide émis par une Autorité de Certification reconnue (ex: Let's Encrypt gratuit via Certbot).
    • Forcez l'utilisation exclusive de TLS 1.2 et TLS 1.3. Désactivez formellement TLS 1.0, TLS 1.1 et les chiffrements obsolètes (RC4, 3DES).
    • {% endif %} {% if detect.headers %}
  • [APPLICATIF] Absence d'en-têtes de sécurité Web :
    Votre application manque de protections contre les attaques clientes (XSS, Clickjacking).
    • Correction : Intégrez les directives suivantes dans votre `.htaccess` ou configuration serveur :
    • `Strict-Transport-Security: max-age=31536000; includeSubDomains` (Force le HTTPS).
    • `X-Content-Type-Options: nosniff` (Empêche le détournement de type MIME).
    • `Content-Security-Policy` (Restreint le chargement de scripts malveillants).
    • {% endif %}
